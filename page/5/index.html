When to use with sharing or without sharing

①Use With Sharing when the User has access to the records being updated via Role, Sharing Rules, Sales Teams - any sort of sharing really.

Apexクラスにwith sharingキーワードを付与します。
これによって、ユーザに適用されているレコードレベルの共有ルールを強制実行することができます（ユーザモードでの動作）。

 

②Without Sharing is reserved for cases where the User does not have access to the records, but there is a need to update them based on user input.

with sharingキーワードを付与しても、オブジェクトのCRUD権限、項目のアクセス権限は、システムモードのままということです。

 

③with sharingキーワードもwithout sharingキーワードも付与しない場合は、呼び出し元のApexクラスの設定が有効となります。
例えば、with sharingキーワードが付与されたApexクラスから、キーワードなしのApexクラスを呼び出した場合、ユーザに適用されている共有ルールが強制実行されます。呼び出し位置が最上位レベルの場合（最初に呼ばれるApexクラスの場合）は、システムモードでの動作となります。

官方文档：Using the with sharing or without sharing Keywords

Use the with sharing or without sharing keywords on a class to specify whether or not to enforce sharing rules.
The with sharing keyword allows you to specify that the sharing rules for the current user be taken into account for a class. You have to explicitly set this keyword for the class because Apex code runs in system context. In system context, Apex code has access to all objects and fields— object permissions, field-level security, sharing rules aren’t applied for the current user. This is to ensure that code won’t fail to run because of hidden fields or objects for a user. The only exceptions to this rule are Apex code that is executed with the executeAnonymous call and Chatter in Apex. executeAnonymous always executes using the full permissions of the current user. For more information on executeAnonymous, see Anonymous Blocks.

Use the with sharing keywords when declaring a class to enforce the sharing rules that apply to the current user.
Use the without sharing keywords when declaring a class to ensure that the sharing rules for the current user are not enforced. For example, you may want to explicitly turn off sharing rule enforcement when a class acquires sharing rules when it is called from another class that is declared using with sharing.

MBR分区表、GPT分区表、ESP分区、MSR分区的简介

1. MBR分区表：
Master Boot Record，即硬盘主引导记录分区表，只支持容量在 2.1TB 以下的硬盘，超过2.1TB的硬盘只能管理2.1TB，最多只支持4个主分区或三个主分区和一个扩展分区，扩展分区下可以有多个逻辑分区。

2. GPT分区表：
GPT，全局唯一标识分区表(GUID Partition Table)，与MBR最大4个分区表项的限制相比，GPT对分区数量没有限制，但Windows最大仅支持128个GPT分区，GPT可管理硬盘大小达到了18EB。只有基于UEFI平台的主板才支持GPT分区引导启动。

3. ESP分区：
EFI system partition，该分区用于采用了EFI BIOS的电脑系统，用来启动操作系统。分区内存放引导管理程序、驱动程序、系统维护工具等。如果电脑采用了EFI系统，或当前磁盘用于在EFI平台上启动操作系统，则应建议ESP分区。

4. MSR分区：
即微软保留分区，是GPT磁盘上用于保留空间以备用的分区，例如在将磁盘转换为动态磁盘时需要使用这些分区空间。

MSR分区即 Microsoft 保留 (MSR) 分区。是每个 GUID 分区表 (GPI) 磁盘都要求的分区。 　　系统组件可以将 MSR 分区的部分分配到新的分区以供它们使用。例如，将基本 GPT 磁盘转换为动态磁盘后，系统分配的 MSR 分区将被用作“逻辑磁盘管理器”(LDM) 元数据分区。 　　MSR 分区的大小会因 GPT 磁盘的大小不同而发生变化。对于小于 16 GB 的磁盘，MSR 分区为 32 MB。对于大于 16 GB 的磁盘，MSR 分区为 128 MB。MSR 分区在“磁盘管理”中不可见，用户也无法在 MSR 分区上存储或删除数据。

Salesforce によるTLS 1.0 無効化対応

問題：

サーバからコードを更新する場合、下記のようなエラー

「この組織ではTLS1.0が無効になりました。HTTPSを使用してSalesforceに接続するには、TLS1.1以降を使用してください」が発生しました。

解決案：

eclipseパスの「eclipse.ini」ファイルに下記の内容を追加する。

-Dhttps.protocols=TLSv1.1,TLSv1.2

Force.com サイトの作成と編集を行う「アプリケーションのカスタマイズ」
Force.com サイトの公開アクセス設定を編集する「ユーザの管理」

Force.com サイトでは、公開 Web サイトとアプリケーションを作成できます。それらは Salesforce 組織と直接統合されるため、ユーザがログインする場合にユーザ名やパスワードは必要ありません。



Force.com サイトを設定する手順は、次のとおりです。

1. [設定] から、[クイック検索] ボックスに「サイト」と入力し、[サイト] を選択します。
2. 組織のカスタム Force.com ドメインを登録します。
3. サイトを作成および設定します。
4. サイトの Visualforce ページを開発して使用し、標準ページを使用するか非表示にし、追加設定なしのエラーページをカスタマイズするか置き換えます。これらのページをサイトと関連付けます。
5. 電子サイトテンプレートを選択します。参照項目を使用して、開発したテンプレートの検索と選択を行うか、用意されているすぐに使用できるテンプレートを使用します。サイトテンプレートには、サイトのページレイアウトおよびスタイルシートが用意され、関連付けられたポータルから継承される書式は上書きされます。
6. ログインまたはセルフ登録するポータルを有効化し、サイトと関連付けます。
7. 必要に応じて、公開アクセス設定を変更します。
8. サイトを公開する準備が整ったら、[サイトの詳細] ページの [有効化] をクリックします。サイトの有効化は、[サイトの編集] ページや [サイト] ページからも行えます。



サイトを設定すると、次のことが実行できます。

サイトに対しレポートおよびダッシュボードを実行する。 サイトに関連する基準を満たす場合にメールアラートを開始するワークフロールールを作成する。

常用的Windows键

常用的 Windows 键，估计很多人还不懂的，这里再提提，让大家学习学习

  1. Windows 键 + 空格键

功能：透明化所有窗口，快速查看桌面。使用此快捷键可以立即将目前所有打开的窗口透明化，以便查看桌面。

  1. Windows 键 + 字母键“D”

功能：最小化所有窗口，并快速查看桌面。这组快捷键能帮助使用者立即将当前桌面上所有窗口最小化，以便查看桌面；再次按下此快捷键时，刚刚最小化的窗口又会全部出现了。

  1. Windows 键 + 数字键

功能：按顺序打开相应程序。与快速启动栏中的程序相对应，如果您能记住快速启动栏中程序的排列顺序，直接按下此组快捷键就能马上调用了。

  1. Windows 键 + 字母键“T”

功能：依次查看已经打开程序的预览图。使用这组快捷键，用户可以从左至右轻松预览已打开程序的缩略图；同时，若使用 Windows键+Shift 键+“T”则可以从右至左反方向查看。

  1. Windows 键 + 字母键“G”

功能：依次显示桌面小工具。如果您的桌面上有太多的小工具，查看起来就不太方便了；使用此组键能迅速让小工具依次显示在面前，想找哪一个都十分方便。

  1. Windows 键 + 字母键“P”

功能：打开“外接显示”的设置面板，当然这个功能只有接入了一些外接设备后才能派上用场。

  1. Windows 键 + 字母键“X”

功能：打开“Windows移动中心”设置面板（这个功能主要面向笔记本电脑用户）。Windows 7 中的移动中心提供了显示器亮度控制、音量控制、笔记本电池状态监控、Mobile 手机同步设置、外接显示器管理…等多种功能。

  1. Windows 键 + 符号键“+” / Windows键+ 符号键“-”

功能：打开放大镜。Windows 7 中新增加了放大镜功能，对于视力不好的使用者来说，这是很好用的一项功能。按下这组键，Windows 7 将自动打开放大/缩小功能。

  1. Windows 键 + 方向键“↑”

功能：使当前使用的窗口最大化。

  1. Windows 键 + 方向鍵“↓”

功能：使当前使用中的最大化窗口恢复正常显示；或者如果当前窗口不是最大化状态，则会将其最小化。

  1. Windows 键 + 方向键“←” / Windows 键 + 方向键“→” (Aero Snap)

功能：使正在使用中的窗口贴向屏幕左侧/右侧，而只占用50% 的显示器面积，熟练使用此功能对同时编辑两个文档并进行校对会很方便。

  1. Windows 键 + Home 键 (Aero Shake)

功能：只显示当前窗口，并且最小化其他窗口，以突出显示当前使用中的窗口。

■WordPress 内存缓存终极加速插件：Batcache

Batcache 工作原理

WordPress 启用 Memcached 这样的内存缓存之后，每次访问 WordPress 页面，都要从内存中获取多个缓存对象。

而 Batcache 主要就是解决这个问题，它是基于 Memcached 的 WordPress 缓存插件，它的工作原理是：把当前整个页面作为一个对象缓存到内存中，这样再次访问 WordPress 页面的时候，直接获从内存中获取这个对象即可，速度极快。

并且这个插件也是 WordPress.com 官方使用的加速插件之一。

Batcache 安装和使用

安装这个插件有点麻烦，不过只要按照以下步骤进行就可以：

  1. 安装好 Memcached（上文所述）
  2. 上传 advanced-cache.php 到 /wp-content/ 目录。
  3. 在 wp-config.php 添加以下代码激活 Batcache:
    define('WP_CACHE', true);

日本东京府中换驾照

■关键点

1.左右转
转之前打灯 左转 看三个1.ルームミラー２．左边ミラー３・车的后排座位左边（因为那里是死角） 之后转过去
右转 一样看三个1.ルームミラー２．右边ミラー３・车的后排座位右边（因为那里是死角） 之后转过去

2.过障碍
提前打灯 确认三个 右转变道 注意过障碍的时候记得 车身要距离障碍50cm以上
然后 打灯 确认三个 左转变道

3.过十字路口
左转 到十字路口前要贴边 （为啥呢？就是为了防后面的小摩托通过） 然后停止
确认绿灯后 左转前 确认三个 转过去

直线通过的时候
如果你是第一辆车，要确认完左右以后才能开过去 而且要慢慢过去

右转 有些不同 因为毕竟对面方向会有来车
所以转的时候 首先打灯变道贴边 之后确认前方有没来车 最后就死盯着车要开去的方向过去就好了

补充下有种十字路口 1.红灯在闪 2.黄灯在闪红灯的时候就当做 止まれ　来对待
黄灯的时候就　减速加上左右确认慢慢地过去就好了

4.S弯和直角转弯
这个在国内驾校有好好学过的都没问题，我就不多说了
唯一就是进去的时候空间要留好，剩下就看感觉了

5.日语
考官是用日语发号施令
所以基本的意思至少要懂吧
还有就是要表现出胸有成竹的样子

6.府中的考场（直角的地方）
有个地方大家容易忽略 这个做不好很容易掉的
很多人就是看到障碍物 右转出去逆道行驶 然后你就差不多可以预约下次了
所以说就是在直角出去的时候 一定要慢速
肯定是右转 然后出去以后会马上碰到障碍物（转出去到左边道以后会在障碍物前面）
这时候就是先慢速转到左边的路上
然后在打右灯确认三个 变到右边路上 然后过了障碍 变到左边路上就行
这个和过障碍一样 就是要一定要慢行

7.关于速度
直线上就加点速度不要超过50，然后转弯要减速 20左右过弯应该可以
考试的时候 大概是 前边可以速度快些 然后到直角S弯的路段尽量要慢速
由于中间路段动作比较多 一个开快就回不来了

8.止まれ
还有就是这个，很多人没停稳就过去了，然后直接不合格
这个考官不会提醒你，眼睛放亮就好
所以动作就是 踩死停稳 然后确认左右边慢慢过去

9.上车和下车
上车 做个样子检查下车前（毕竟是个死角）
调座位 调镜子 安全带 发动 踩煞车 挂挡 拉手闸
接下来的动作要注意 我那天考蛮多人没做的
打右灯 确认三个 没车 才开出去
而不是直接就开出去了

下车 不是直接开门出去
而是小开门一下 头从门缝看出去确认 然后才开门出去
这个考官一定会看 没做的人都被说了

10.找个地方学
最后的建议就是找个地方学
不然看了我说的也是纸上谈兵

 

■过程说明

1.安全确认.很多人挂在这了.确认包括,
〔上车前确认,关门前确认,关门后确认.开信号灯前确认,打方向盘前确认.止まれ线停止,发动前的左右确认〕
2.交通标志啥的要看懂.车是往右开的,有看到 止まれ 要一时停止,等〕
3.直角弯.〔俺第一次就挂这了.等等要重点说下〕
3.S弯道.〔在中国有认真考过驾照的,不算难〕
4.开车的速度,〔有的地方要加速的,要开个30-40这样,很多人也挂这了〕
5.开车的一些不良习惯,〔比如坐姿,反向盘把握程度,,等等〕

府中的路考地图!

这就是府中考场的地形图,
现在我来讲解下.考试的路线及考试该注意的安全确认和路标

考试分有两种路线,一种就是以上,大家看到红色线的路线!
还有一种，

还有红色，我画了圈圈标了123456789…这些是注意点，我会慢慢讲解.
蓝色的路线,其实跟红色路线一样.都会在4的那个口进来.
就是蓝色的多了个过障碍物!

对照俺画的图,
 咱们先从蓝色线路开始,!〔补充.考试时.按顺序排,第一个人考的时候,第2个人可以坐车后面,以此内推〕

首先在起点.上车时,要做以下确认:〔补充下,上车的时候,先把登陆证拿出来〕
1.走到车后面,蹲下来看看有没其它东西〔比如 猫啊,狗啊..等〕
2.从里面再走到车前,蹲下来,同1一样确认..〔别往外面走到车前,.有车什么危险〕
3.确认完毕后.准备往车门走之前,之前.先停一下,确认有没车开过来
4.确认没车开过来后,走到车门.再确认一遍有没车开来.然后打开车门.!
5.人进了车,门别一下子就关上了…门关到.大概只剩15厘米左右时,眼睛从门缝里向外看下.再确认！

以上步骤是,上车前确认,要记住,!

到了车里.〔补充:先把登陆证给副驾驶座的考官,他会跟你核对姓名啥〕
1.门手那边有个锁门的按一下,
2.调座位.绑安全带
3.调调中间的后视镜,做个样子就好.再看看两边的后视镜,一般不用调!
4.考官会问你准备好了没.好的话.钥匙转一下,发动起来.
5.然后别忘了,要踩住刹车,然后再挂到D档,
6.松手刹,.一定要确认清楚手刹是不是拉下去了.
〔有的人手刹没松到位,开动起来.就尴尬了..〕
7.中间后视镜确认,.打右方向灯,右边后视镜确认,在扭头后面确认…然后启动了..
〔关于确认后面会细说〕

然后大家看看俺画的地图及数字标注

从起点开始.到1的那个位置.要停下来.
为啥?有止まれ线呗,,这个教官也会跟你说.
关键停下来要做啥?
左右的确认,.确认完毕,听教官指挥,.

教官会说,下个路口,.〔俺红圈标的2位置〕停下来.
那里也有个止まれ,停下来.左右确认,有没车.
确认完毕听教官指挥,OK.就可以开了.

注意事项,因为蓝色路线是往左边路线开的,所以车往左边靠.停到止まれ上
方向灯,确认啥的.别忘了..

接着转个弯就可以看到障碍物了.〔那个单一的转弯不需要打方向灯〕

过障碍物,30米前,要进行以下确认:
1.先中间后视镜确认.后方有没车
2.打右方向灯.
3.右边后视镜确认,
4.扭头往右边看后面,因为有些死角是镜子看不到的,
以上这些步骤按顺序做完后.可以右转,过障碍物了,
过完障碍物,再1.中间后视镜确认,2.打左方向灯,3.左后视镜确认,.4.左回头确认,然后左转,回到左道上.

以上这4点我是按顺序排的,为啥?
因为再要改变方向,打出方向灯时要,要给后方车辆3秒左右的反应时间.
1先后视镜确认,是先看后方有没车.然后打出方向灯,再进行,3.4的确认.
3.4确认完毕后,时间也差不多3秒了,你就可以转弯了..变方向了..

〔 这图不是教你怎么超车,,,是把黄色那车イメージ成障碍物〕

所有安全确认中,需要打方向键,都要以上的确认步骤,,
单一的路线,只转方向盘就只要,1.3的步骤,,,就不需要回头看了….

做这些步骤,一定要发自内心的去做,让它成为一种习惯,,这样对以后驾车很有帮助的,.!

然后就是打方向灯,确认,进入那个弯道
在车开到我画的那个红圈4的时候.教官会跟你说哪个口进入.〔要做直角弯了〕
这时差不多要变更方向了.30米前.确认.方向灯,向右靠.
看那图,红圈4-5之间的红色路线.是不是斜着过来,靠直角弯方向靠了?
然后准备弯进去的时候,放慢速度,在做下,打方向盘前的确认.!

接着进入直角弯,
俺第一次就挂在这了..原因是后轮离那角太近了,..
以下图来表示:

画不好别见笑,
进入直角弯靠右边开,.靠左的话,后轮就会碰到那个角,
〔虽然说碰到那角,在轮胎还没落地前,可以倒车,不过能不碰还是别碰最好〕
靠中间开的,就会犯跟我一样的错误,后轮离角太近了.也算不合格!
所以靠右,,然后那车里小人的视线.跟那角 快要 平行的时候.大胆转过来呗.
剩下的就自己大脑里イメージ下!

直角出来的时候,也别忘了停一下左右确认打方向灯啥,,,\
出来后,.会看到红圈6有个,黑色的小障碍物..
没什么,别碰到就好..方向灯该打的打,,该补方向灯的要补!!

后面就是S弯道了,,能开始跑S弯道,说明你离拿驾照不远了..
因为很多不合格直角开完,就直接回来了..

沿着那边慢慢滑过去..然后摆正,往前开,,再切到中间点.才顺着边沿滑过去…
这个俺就不画图了..
脑海里,イメージ一下呗,中国考驾照时,也有教的!!

S弯出来后,.问题就不大了..唯一要注意的地方就是红圈11号那的止まれ线.
 线不是重点,重点是头上的信号灯,,很多人没注意到上面的信号灯..
信号灯红色的时候,你就老实停着吧,,当变黄的时候.左右确认下就可以开了…

然后红圈13.别忘了向右变更!
接着开到终点了.也是起始点.
差不多就这样了.!!

现在俺总结几个要点:

1.确认确认,再确认,..千万别怕麻烦,确认别都只做个样子!!

30米变更大概就是这图的感觉,,
3.很多直线,要记的加速,速度提到30.40左右.快进入转弯时候在慢慢减速
