(3)ILMT Agent Install on AIX, ILMT agent安装

[shell]

对于支持使用 ILMT_INSTALL_PATH 环境变量的安装，请设置变量以指向定制位置：
export ILMT_INSTALL_PATH=PATH_TO_DESIRED_LOCATION

对于支持使用基于用户指定的安装位置 (USIL) 的重定位机制的安装，请执行以下步骤：
创建 custom_location 目录：
mkdir -p /custom_location

将该目录设为用户指定的安装位置 (USIL)：
mkusil -R /custom_location

在此位置中安装代理程序：
installp -R /custom_location -acgXd full_path_to_the_installation_package_dir ILMT-TAD4D-agent-7.5-aix-ppc ILMT-TAD4D-agent

代理程序安装在 /custom_location/opt/itlm 和 /custom_location/var/itlm 文件夹中

[/shell]

参考资料

(4)lslpp指令找不到ILMT agent？？

由于通过了mkusil -R指定了安装目录，会导致lslpp指令找不到ilmt的相关信息。

通过lslpp指定目录，执行下面指令就能正常显示了。

[shell]</pre>

#lspp -R /export/appl/pkgs/itlm -al
<pre>[/shell]

（5）一个错误信息的处理

[shell]

#installp -R /opt/ilmt -acgXd ILMT-TAD4D-t -acgXd ILMT-TAD4D-agent-7.5-aix-ppc ILMT-TAD4D-agent
+—————————————————————————–+
Pre-installation Verification…
+—————————————————————————–+
Verifying selections…installp: A system error occurred while attempting
to access the Software Vital Product Data.
Use local problem reporting procedures.
#
通过执行rmusil后重新执行安装指令

#rmusil -R /opt/ilmt
#
[/shell]
